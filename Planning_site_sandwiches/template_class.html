Brief

This is a Sandwich ordering program with which the user can select from multiple options of different functions. These include printing out the sandwich menu with prices, printing out the order menu (the items the user has added to their order), adding an item to their order along with the quantity, editing the order list, this editing include the changing of quantity, deleting an item as a whole, or canceling this function incase it was a mistake. More options include, deleting the order list and starting a new order and entering the customers details which include a dilivery or pickup option, getting the users name and phone number and the dilivery address if they chose the dilivery option. The finial function they can choose is confirming the order - this confirms the order list and the customer details. Finally, there is a quit option incase the user wishes to quit the program.

Relevant Implications - described:

  • Functionality: Functionality has to do with if the program works properly. So making sure the program doesn't crash at any point or there aren't any bugs, for example; adds up price correctly.
  • Usability: Usability is making sure that the user can easily operate the program/use it. It's also about whether the user can follow a logical process to make a sandwich order and be able to view their order details and details in general. Usability is about asking if the menu options are easy to understand and can lead the user through the processes with clear communication. If the user makes a mistake clear usability will give the appropriate feedback and ask them to try again. xGood usability is if the user is able to view their order status and their details in a clear, logical, and easy way, can they also change their mind and edit certain details.
  • Future proofing: Future proofing is ensuring that the program is easy to main and whether another programmer can quickly understand how it works. Future proofing is also about whether a new feature cna be added easily and if the code follows a logical structure for this to happen.
